Algorithmic LBT

LBT is the "last bottom triple" part of solving the face-turning octahedron using the Bencisco method

Traditionally LBT is solved on the top layer by using a series of flipping algs along with U moves, then inserted in the bottom layer. In some cases this is easy, but in other cases it's not, requiring a couple flips and extra setup moves to take the corner or triangles out of the bottom slot.

As an alternative, the alg set in the first section aims to solve the LBT with the corner already in place on the bottom (flipped or not) with a single algorithm. There are 31 main cases (plus the already-solved case), organized into eight groups. Each group has one case, its mirror case, and those same cases with the corner flipped. Sometimes U or Uw setup moves are needed to put the relevant triangles into place on the top/middle layers. Usually multiple algs can be applied, since there are two of each side's triangles that can be used. This means some algs/groups are redundant (I personally don't use groups 7 or 8 at the moment)

Additionally, some algs for cases with the corner on top are now included. More may be added later.

These algs were all generated by myself using custom software and hand-picked for ergonomics (except the four keyhole cases, discovered by 360cubed). They are somewhat optimized for right-hand dominant solving, but since each case has a mirror, all cases can use the mirrored alg of the mirrored case.

Note: the notation of these algs solves the triple on the bottom left, but the diagrams show the left face in the front for recognition.
