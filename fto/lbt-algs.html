Algorithmic LBT

LBT is the "last bottom triple" part of solving the face-turning octahedron using the Bencisco method

Traditionally LBT is solved on the top layer by using a series of flipping algs along with U moves, then inserted in the bottom layer. In some cases this is easy, but in other cases it's not, requiring a couple flips and extra setup moves to take the corner or triangles out of the bottom slot.

As an alternative, this alg set solves the LBT with a single alg ("one look"), no matter where the corner or triangles are, though possibly needing U and/or Uw setup moves first. There are 95 cases (plus the already-solved case), organized into six sections, which are further subdivided into groups of four cases. Each group has one case, its mirror case, and those same cases with the corner flipped. Generally, multiple algs can be applied for any given configuration of the FTO, since there are two of each color's triangles that can be used (the only exception is cases in group 3: if both triangles of one color are on the top, both of the other color are in the middle, and the corner is on the bottom, only one alg is possible). This means some cases are redundant. The minimal number of algs required for a complete one-look LBT is 60 algs, though these algs have to be chosen carefully to not overlap.

These algs were mostly generated by myself using custom software and hand-picked for ergonomics (except the four keyhole cases [1, 2, 5, 6], discovered by 360cubed, the "trapped triangles" cases [33, 34, 37, 38] which were discovered by Ben Streeter, and some simple cases that use traditional flipping algs, which were also discovered by Ben Streeter). They are somewhat optimized for right-hand dominant solving, but since each case has a mirror, all cases can use the mirrored alg of the mirrored case for left-hand solving.

Note: the notation of these algs solves the triple on the bottom left, but the diagrams show the left face in the front for recognition.
